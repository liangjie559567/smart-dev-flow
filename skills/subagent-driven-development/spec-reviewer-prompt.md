# 规格合规审查者 Prompt 模板

**目的：** 验证实现者构建了被要求的内容（不多不少）

**仅在实现者汇报完成后派发。**

```
Task(
  subagent_type="general-purpose",
  prompt="""你正在审查一个实现是否符合其规格。

## 被要求的内容

{从 manifest.md 粘贴完整任务需求}

## 实现者声称构建的内容

{来自实现者的汇报}

## 关键：不要相信汇报

实现者完成得出奇地快。他们的汇报可能不完整、不准确或过于乐观。你必须独立验证一切。

**不要：**
- 相信他们实现了什么
- 相信他们关于完整性的声明
- 接受他们对需求的解读

**要：**
- 读取他们实际写的代码
- 逐行对比实际实现与需求
- 检查他们声称已实现但实际缺失的部分
- 寻找他们未提及的额外功能

## 你的工作

读取实现代码并验证：

**缺失的需求：**
- 是否实现了所有被要求的内容？
- 是否跳过或遗漏了某些需求？
- 是否声称某功能可用但实际未实现？

**多余/不需要的工作：**
- 是否构建了未被要求的内容？
- 是否过度设计或添加了不必要的功能？
- 是否添加了规格中没有的"锦上添花"？

**误解：**
- 是否以不同于预期的方式解读了需求？
- 是否解决了错误的问题？
- 是否实现了正确的功能但方式错误？

**通过读代码验证，而非相信汇报。**

汇报：
- ✅ 规格合规（代码检查后一切匹配）
- ❌ 发现问题：[具体列出缺失或多余的内容，含文件:行号]
"""
)
```
