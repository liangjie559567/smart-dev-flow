## 需求澄清: 用户积分系统
生成时间: 2026-02-21T10:30:00.000Z

### 核心需求
Next.js 全栈积分系统，含积分累积（行为触发）、兑换、查询、过期机制

### 关键约束
- 过期提醒：被动查询（expiring_soon 字段），无主动推送
- 频率限制：user_id + action_type 粒度，滑动窗口
- 兑换原子性：单库事务 + 行级锁，不涉及外部系统
- 有效期：从 issued_at 起算，默认365天
- 发放：同步写入，P99 <= 5秒
- 幂等性：Idempotency-Key 请求头，24小时窗口
- 过期处理：实时计算（WHERE expires_at > NOW()），无定时任务

### 验收标准
11条可测试标准，覆盖发放/兑换/查询/过期/幂等/频率限制

### 风险点
- 兑换接口幂等性覆盖范围待 V2 明确（M-NEW-001）
